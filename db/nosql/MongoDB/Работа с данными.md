## Добавление данных
Для добавления документа в коллекцию используется `insertOne()`
```mongosh
test> show dbs
admin   100.00 KiB
config   72.00 KiB
local    64.00 KiB
test> show collections

test> db.users.insertOne({"name": "Tom", "age": 28, "languages": ["english", "spanish"]})
{
  acknowledged: true,
  insertedId: ObjectId('694956f80be02b50f63f118c')
}
test> show collections
users
test> show dbs
admin   100.00 KiB
config   36.00 KiB
local    64.00 KiB
test      8.00 KiB
test> 
```
Пример выше показывает, что изначально у меня было 3 базы данных
- `admin`
- `config`
- `local`
Но при этом выбрана база данных `test`, которой нет физически. Это одна из особенностей MongoDB. Она создает базу данных лениво (lazy creation) и пока в ней не появится запись, она будет создана виртуально (логически). Поэтому при добавлении (в нашем случае создании и добавлении) коллекции `users`, создается новая физическая база данных `test` с коллекцией `users` и объектом с `_id=694956f80be02b50f63f118c`. 
Примечание: Названия ключей могут использоваться в кавычках, а могут без них. Следующие записи считаются идентичными:
```mongosh
db.users.insertOne({"name": "Tom", "age": 28, "languages": ["english", "spanish"]})
db.users.insertOne({name: "Tom", age: 28, languages: ["english", "spanish"]})
```

При добавлении документов в коллекцию можно явно указать значение идентификатора, например `_id: 1234`, или использовать тип данных ObjectId, в таком случае ключ должен иметь размерность в 12 байт (24 символа) `_id: ObjectId("abcdefabcdefabcdefabcdef")`. Примечание: тип данных для поля `_id` может быть целым числом, строкой или объектом ObjectId, в который могут входить символы a, b, c, d, e, f в нижнем и верхних регистрах, а также цифры 0-9 и длина должна быть 24 символа.
```mongosh
test> db.users.insertOne({"_id": ObjectId("abcdefabcdefabcdefabcdef"), "name": "John", "age": 31, "languages": ["turkish", "spanish"]})
{
  acknowledged: true,
  insertedId: ObjectId('abcdefabcdefabcdefabcdef')
}
```
Чтобы убедится, что объекты были добавлены в коллекцию, можно использовать функцию `find()`, которая по умолчанию выводит все объекты коллекции:
```mongosh
test> db.users.find()
[
  {
    _id: ObjectId('694956f80be02b50f63f118c'),
    name: 'Tom',
    age: 28,
    languages: [ 'english', 'spanish' ]
  },
  {
    _id: ObjectId('abcdefabcdefabcdefabcdef'),
    name: 'John',
    age: 31,
    languages: [ 'turkish', 'spanish' ]
  }
]
```
Чтобы добавить несколько документов, модно использовать функцию `insertMany()` и передать массив с документами для вставки в коллекцию:
```mongosh
test> db.users.insertMany([{"name": "Bob", "age": 26, "languages": ["english", "french"]}, {"name": "Alice", "age": 31, "languages":["german", "english"]}])
{
  acknowledged: true,
  insertedIds: {
    '0': ObjectId('6949635e0be02b50f63f118d'),
    '1': ObjectId('6949635e0be02b50f63f118e')
  }
}
```
Также поддерживается создание документа `document=({...})` с последующим добавлением, т.е. в 2 этапа:
```mongosh
test> document=({"name": "Bill", "age": 32, "languages": ["english", "french"]})
... db.users.insertOne(document)
{
  acknowledged: true,
  insertedId: ObjectId('694963f90be02b50f63f118f')
}
```
Данные для базы данных mongodb можно определять в обычном текстовом файле, что довольно удобно, поскольку мы можем переносить или пересылать этот файл независимо от базы данных mongodb. Например, определим где-нибудь на жестком диске файл `users.js` со следующим содержимым:
```js
db.users.insertMany([
{"name": "Alice", "age": 31, "languages": ["english", "french"]},
{"name": "Lene", "age": 29, "languages": ["english", "spanish"]},
{"name": "Kate", "age": 30, "languages": ["german", "russian"]}
])
```
Теперь, если поместить данный файл в папку `scripts`, то можно будет выгрузить содержимое данного файла в коллекцию `users`:
```mongosh
test> load('scripts/users.js')
true
```
